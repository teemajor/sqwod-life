# Sqwod Intelligence — fact bank

Auto-extracted candidate data from the private Statista PDFs in `research/statista/`.
Generated by `automation/intelligence.py`. The PDFs stay private (gitignored);
these extracted **facts + provenance** are committed and power the living
Intelligence reports.

## What this is — and isn't

This is a **triage / discovery layer**, not ground truth. Statista's PDF text
interleaves axis ticks and stacked multi-series data without structure, so fully
automated parsing is unreliable. The extractor surfaces:

- which studies and charts exist (title, unit, year axis, source attribution)
- candidate numeric series (`series_rows` holds every row it found)
- a `needs_review: true` flag whenever a chart is multi-series or the value count
  doesn't match the year axis

**Before any number enters a published report, confirm it against the source page.**
The flagship "State of Wearables" report uses hand-verified figures pulled directly
from study 166600's chart pages.

## Workflow

1. Drop new Statista PDFs into `research/statista/`.
2. `python3 automation/intelligence.py` → regenerates this fact bank.
3. Use it to find the chart you need, confirm the exact series against the PDF,
   then drop verified `figures` / `series` into the report's frontmatter and add a
   `changelog` entry.

## Refresh cadence (living reports)

Statista's own outlooks update biannually (Market Outlooks) to annually (Industry
Reports). Our living reports refresh faster so they stay the freshest cited source
in the niche:

- **Quarterly** — flagship market reports (State of Wearables, etc.): re-pull
  figures, log the change, bump `updatedAt`.
- **Monthly** — money-movement / momentum figures that move fast.
- **On new data** — whenever a fresh Statista study or a stronger public figure lands.

Each refresh = a new `changelog` entry on the report, so readers (and search
engines) see a maintained, dated, cited document — the moat vs. one-and-done posts.
